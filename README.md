# Odin Recipes Project
## 📌 Project Overview
This is the first major project in The Odin Project's Foundations curriculum. The goal of this project is to build a basic recipe website from scratch using fundamental HTML. It serves as a practical playground to transition from learning theories to writing actual code locally and managing it via Git.

## 🚀 Objectives: What I Will Be Doing
In this project, I will be creating a multi-page website consisting of a main index page and at least three distinct recipe pages.

To accomplish this, I will be implementing the following core concepts:

    1. Command Line Basics: Initializing the project repository, navigating directories, and managing files entirely through the terminal.

    2. HTML Boilerplate: Setting up proper, valid document structures with <!DOCTYPE html>, <html>, <head>, and <body> tags.

    3. Working with Text: Structuring recipe content logically using headings (<h1> to <h3>) and paragraphs (<p>).

    4. Lists: Using unordered lists (<ul>) for ingredients and ordered lists (<ol>) for sequential cooking steps.

    5. Links: Implementing anchor tags (<a>) to connect the main page to the individual recipes and back.

    6. Images: Embedding relevant images for each recipe using the <img> tag and ensuring appropriate alt attributes are configured.

    7. Git & GitHub: Tracking my progress with atomic commits and pushing the final project to a remote GitHub repository.

## 🛠️ Tech Stack & Tools
Language: HTML5

Version Control: Git & GitHub

Editor: VS Code

Terminal: Bash / Command Line

## 🔮 What I Expect to Gain
By building this project before moving on to CSS, I aim to solidify my understanding of raw HTML document flow. It will help me build muscle memory for structuring clean, semantic markup and establish a confident Git workflow early on.

## 📝 What I Learned (Post-Project Reflection)
Having successfully completed and deployed the project, here are my key technical takeaways:

    1. File Paths & Directory Structure: I learned the critical difference between relative and absolute file paths. Ensuring that image sources (img/) and internal page links (recipes/) point to the correct relative directories was essential to prevent broken links on GitHub.

    2. Semantic Hierarchy: I practiced structuring content strictly using HTML semantics. I discovered how using <h1> for the main title, <h2> for major sections (like Ingredients and Steps), and <h3> for smaller sub-headings keeps the document clean and accessible.

    3. Lists Integration: Implementing <ul> for the bulleted shopping lists and <ol> for the ordered, step-by-step cooking procedures helped me understand how browser formatting handles sequential versus non-sequential data.

    4. Git Workflow Confidence: Git is no longer just a theoretical concept. By breaking my work down into atomic commits with descriptive messages, I built genuine muscle memory for initializing, staging, committing, and pushing code directly from the terminal.
